# SITE_MAP.md

## Scope and Access Model

This document distinguishes routes implemented in the current application from planned or optional future routes. Future routes are not current launch commitments unless `docs/strategy/PUBLIC_LAUNCH_READINESS_CHECKLIST.md` explicitly adds them to launch scope.

All currently implemented pages and capability modules are open for direct exploration. Orientation guides engagement and explains the system; it does not unlock content or create a completion prerequisite.

## Implemented Page Routes

- `/` — Landing page
- `/about` — About the project / creator POV
- `/course` — Course overview hub
- `/course/orientation` — Intro module
- `/course/dashboard` — Learner progress dashboard
- `/course/module/[slug]` — Individual module pages

## Implemented API Routes

- `/api/reflection-feedback` — Reflection feedback endpoint
- `/api/scenario-feedback` — Scenario feedback endpoint

## Planned or Optional Future Routes

- `/course/module/[slug]/experience` — Optional deep-linked interaction view
- `/course/module/[slug]/reverse-engineering` — Optional focused Reverse Build breakdown; route name may remain technical unless intentionally renamed later
- `/course/resources` — Supporting resources / references / downloads
- `/course/completion` — Course completion summary

These routes describe possible future organization. They are not implemented routes or current access requirements.

## Page-Level Flow

### Landing Page

- hero
- project positioning
- why this matters now
- preview of Bradley's current seven-capability pathway as the first complete system demonstration
- CTA: start the course

### Course Overview

- course promise
- audience
- learning objective
- module map
- estimated completion time
- progress model explanation

### Orientation Module

- why instructional design is changing
- Bradley's professional-calibration rationale for selecting the current seven-capability pathway, with research supporting individual capability concepts rather than selecting or validating the exact collection
- explanation of the reusable method and learning architecture
- explanation of how to use the guided, open-access experience
- optional progress completion action
- open choice to enter any capability module

### Module Page Layout

1. module header
2. capability overview
3. why demand is rising
4. L&D model connections
5. core content sections
6. interactive experience embed/launch
7. Reverse Build section preview
8. reflection/checkpoint
9. complete module CTA
10. next module CTA

### Dashboard

- learner progress bar
- modules completed
- current module
- recommended next step
- revisit completed modules

### Planned Completion Page

- course completion status
- summary of Bradley's current seven-capability pathway
- reflection prompt
- next actions
- optional share-on-LinkedIn CTA

The planned completion page must not become a prerequisite for access or imply that Bradley's current pathway permanently defines the reusable system.
