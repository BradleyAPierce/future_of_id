# Future of ID — Learning Framework Library

## Purpose

This document defines the Future of ID Learning Framework Library.

The library captures the enduring mental models that shape the product's visual language, learner experience, and professional philosophy. These frameworks should help learners understand what kind of instructional design work Future of ID is preparing them to do.

This is not an implementation document, design-system document, or image inventory. It is a product philosophy document.

## Document Ownership

This document owns the conceptual purpose of Future of ID's reusable learning frameworks.

Use it when deciding whether a framework deserves to become a reusable artifact, how it should shape the learner's thinking, and what emotional or professional shift it should support.

Implementation details, filenames, component behavior, and image placement belong elsewhere.

## Library Philosophy

Every framework should:

- teach one enduring mental model
- answer one transformational question
- reduce cognitive or emotional friction
- become a reusable learning artifact
- remain valuable regardless of future technology changes

A strong framework should make the learner feel more oriented, not more impressed. It should compress complexity into a useful way of seeing.

## Frameworks

### Learning System

- Framework Name: Learning System
- Transformational Question: How do I move from creating course assets to designing connected learning systems?
- Mental Model: Future-ready instructional design connects learning goals, practice, feedback, technology, evidence, and professional judgment into one system.
- Emotional Job: Help learners feel oriented inside a field that can otherwise feel scattered by tools, trends, and competing advice.
- Core Philosophy: Instructional designers are not merely content producers. They are system designers who shape conditions for learning and performance.
- Visual Metaphor: A connected system with multiple reinforcing parts.
- Intended Uses: Homepage, orientation, presentations, downloads.
- Memorable Line: Design the system, not just the lesson.

### Experience in Action

- Framework Name: Experience in Action
- Transformational Question: How does a learner practice judgment and then understand why the experience worked?
- Mental Model: The learner moves through experience, AI-supported feedback, reflection, and Reverse Build so practice and design rationale reinforce each other.
- Emotional Job: Help learners trust that Future of ID teaches through doing, not through explanation alone.
- Core Philosophy: The most valuable learning moments come when experience, feedback, and design transparency work together.
- Visual Metaphor: A loop between learner action and visible design rationale.
- Intended Uses: Homepage, orientation, modules, presentations.
- Memorable Line: Practice first, then inspect the design.

### Different Paths

- Framework Name: Different Paths
- Transformational Question: What future-ready instructional design path fits my role, context, and level of technical ownership?
- Mental Model: Instructional designers can grow through multiple credible paths rather than one prescribed technical ladder.
- Emotional Job: Reduce anxiety that every instructional designer must become the same kind of technologist.
- Core Philosophy: Future-ready practice is not one destination. It is a set of strategic choices shaped by context, interest, and responsibility.
- Visual Metaphor: Multiple routes across the same professional landscape.
- Intended Uses: Homepage, orientation, downloads, presentations.
- Memorable Line: There is more than one credible future.

### Capability Ecosystem

- Framework Name: Capability Ecosystem
- Transformational Question: How do the seven capabilities connect into a stronger instructional design practice?
- Mental Model: The seven capabilities are not isolated topics. They form an ecosystem of judgment, evidence, systems thinking, learner support, and human-centered change.
- Emotional Job: Help learners see the course as a coherent professional map rather than a list of modules.
- Core Philosophy: Capabilities become powerful when learners understand how they influence one another.
- Visual Metaphor: An ecosystem of connected capabilities.
- Intended Uses: Orientation, dashboard, modules, downloads, presentations.
- Memorable Line: Capabilities connect before they compound.

### Reverse Build

- Framework Name: Reverse Build
- Transformational Question: What can I learn by inspecting the design choices behind the experience?
- Mental Model: Reverse Build reveals the rationale, constraints, tradeoffs, and human judgment behind a learning experience.
- Emotional Job: Make learners feel invited behind the curtain instead of left to guess how the product was designed.
- Core Philosophy: Transparency builds trust and helps learners transfer design judgment into their own work.
- Visual Metaphor: Opening the build to reveal the decisions inside.
- Intended Uses: Modules, experience pages, downloads, presentations.
- Memorable Line: The design rationale is part of the lesson.

### Human Judgment in the Age of AI

- Framework Name: Human Judgment in the Age of AI
- Transformational Question: What should instructional designers own when AI can generate more of the work?
- Mental Model: AI can accelerate analysis, drafting, feedback, and iteration, but human judgment remains responsible for purpose, ethics, context, quality, and consequence.
- Emotional Job: Replace fear or hype with a grounded sense of professional agency.
- Core Philosophy: AI does not replace instructional designers. It raises the importance of clear judgment, learning science, systems thinking, and responsible decision-making.
- Visual Metaphor: Human judgment as the center of an AI-supported design environment.
- Intended Uses: Homepage, orientation, modules, AI interactions, presentations.
- Memorable Line: AI accelerates the work; judgment gives it direction.
