# Future of ID — Situational Mental Models

## Purpose

Situational Mental Models are compact decision aids that help instructional designers make one professional judgment well.

Unlike the Learning Framework Library, they are not broad product philosophies. They are practical thinking tools used within specific learning contexts.

Use Situational Mental Models when a learner needs a simple way to reason through a recurring design decision without adding unnecessary theory or visual weight.

## Core Definition

A Situational Mental Model is a compact, reusable decision aid that helps instructional designers make one professional judgment more effectively.

Unlike Learning Frameworks, which reshape how learners understand the profession, Situational Mental Models support a specific design decision within a particular context.

They reduce cognitive load, reinforce professional judgment, remain useful beyond the course, and are simple enough to become practical job aids.

## Relationship to the Learning Framework Library

Learning Frameworks answer:

WHY

They reshape professional identity.

Situational Mental Models answer:

HOW

They support one professional decision.

Examples:

Human Judgment
-> Why judgment matters.

Evidence -> Interpretation -> Decision -> Measurement
-> How to make an evidence-based design decision.

## Characteristics

A good Situational Mental Model:

- answers one professional question
- supports one decision
- reduces cognitive load
- fits on one screen
- survives technology changes
- can become a practical job aid
- eventually disappears into professional practice

## Design Principles

- One mental model per module maximum.
- Do not create a mental model if an existing Learning Framework already solves the problem.
- Mental models should simplify decisions, not summarize theory.
- Use verbs and decision flow where appropriate.
- Mental models should reinforce professional judgment rather than prescribe one correct answer.

## Future Expansion

New Situational Mental Models should only be created when they solve a recurring professional decision that is not already addressed by an existing Learning Framework.

Avoid creating large collections of diagrams. Quality is more important than quantity.

A useful Situational Mental Model should feel like a decision tool the learner might use again at work, not a diagram added to make a module feel more visual.

## Implementation Guidance

Modules may use Situational Mental Models sparingly when they reduce cognitive load and improve professional decision making.

They should appear only when they help the learner make a clearer design decision in context. If a mental model merely restates the lesson, repeats an existing Learning Framework, or adds visual weight without improving judgment, it should not be used.
